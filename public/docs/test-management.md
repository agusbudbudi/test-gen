# Test Management

An integrated suite to manage your test cases, organize them into folders, and track execution via Test Runs.

## Overview
**Test Management** is the central hub for your QA assets. It allows for a hierarchical organization of test cases and provides a dedicated space for executing tests and tracking results.

## Key Components

### 1. Test Cases Tab
This is your library of test scenarios.
- **Tree View**: Use the left sidebar to navigate through folders and files.
- **Folder Management**: Create, rename, move (via drag-and-drop), and delete folders to organize your projects.
- **Test Case Editor**: Click on a test case to edit its details:
    - **Section & Title**
    - **Case Type & Tags**
    - **Preconditions, Steps, and Expected Results**
- **Bulk Actions**: Select multiple items to move or delete them at once.

### 2. Test Runs Tab
Track the execution of your test cases.
- **Create Test Run**: Start a new execution cycle.
- **Execution Status**: Mark cases as `Passed`, `Failed`, `Blocked`, or `Untested`.
- **Results Tracking**: Keep a history of your release testing cycles to ensure quality over time.

## How to Use

### Organizing Your Library
- Use **Folder+** to create a new project or section.
- You can drag test cases into folders or move entire folders to restructure your library.

### Adding Test Cases
- Use **File+** to create a manual test case.
- Remember that you can also save cases directly from **AI Generate TCs** and **AI Reviewer** into these folders.

### Searching
- Use the **Search Bar** in the tree view to quickly find test cases by title, ID, or tag. The tree will automatically expand to show relevant matches.

## Tips
- **Consistent Tagging**: Use tags like `Smoke`, `Regression`, or `Critical` to filter your cases effectively during search.
- **Version Control**: Keep your test cases updated as requirements change via the inline editor.
