# AI Test Case Reviewer

Improve the quality of your existing test cases with AI-driven analysis, gap detection, and automated improvements.

## Overview
The **AI Test Case Reviewer** analyzes your test cases to identify strengths, weaknesses, and missing scenarios. It provides an "Improved Version" of your test cases, ensuring better coverage and clarity.

## How to Use

### 1. Provide Test Cases
- **Paste Text**: Paste your test cases directly into the main text area (TSV or plain text format).
- **Upload File**: Click **Upload File** to load test cases from an Excel (.xlsx), CSV, or TSV file.
- **Dynamic Import**: You can also send test cases here directly from the [AI Generate TCs](/generate) page.

### 2. Run Review
- Click **Review Test Case**. The AI will evaluate:
    - **Summary**: A high-level assessment of the test quality.
    - **Strengths**: What is already well-covered.
    - **Suggestions**: Specific advice on how to improve clarity or logic.
    - **Missing Cases**: Scenarios you might have overlooked (e.g., edge cases, error handling).

### 3. Review Improvements
- Scroll down to see the **Improved Version** table.
- AI-refined cases will be marked with status indicators to show what was changed or added.

### 4. Save & Export
- **Save to History**: Keep a record of the review in your [History](/history).
- **Copy Improved**: Copy the refined cases to your clipboard.
- **Add to Folder**: Save the improved cases to your [Test Management](/test-management) library.

## When to Use This Feature
- Before a major release to ensure test coverage is robust.
- When peer-reviewing test cases created by other team members.
- To learn better testing patterns by observing AI suggestions.
